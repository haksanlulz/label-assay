Check many labels

For a batch of applications — up to {{ max_files }} files or {{ max_mb }} MB per upload, each image 5 MB or less; a full 300-application drop fits in one go. Add the application data as a CSV and each label is checked against its own application row, exactly as it would be one at a time. Results appear as they finish.

Columns: filename, brand_name, class_type, and optionally fanciful_name — when a fanciful name was filed, the label may display either name and the brand check accepts both. Without the CSV, every label-internal check still runs — only the brand comparison is skipped, since there is nothing filed to compare against.

Some labels print the government warning sideways along an edge. When the warning is not found upright, the label is re-read rotated — up to three extra read passes for that label only. Uncheck for maximum throughput.
